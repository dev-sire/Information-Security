Information Assurance Principles

  • Confidentiality: User passwords are hashed using bcrypt before storage, ensuring that even if the database is compromised, the passwords remain secure.
  • Integrity: Data integrity is maintained through secure hashing algorithms and validation checks during registration and login.
  • Availability: The system is designed to be responsive and available to users, with error handling to manage potential issues.
  • Non-repudiation: User actions are logged, ensuring that users cannot deny their actions within the system.

Security Overview

Assets

User data (username, hashed passwords), application code, and database.

Threats

Unauthorized access, data breaches, SQL injection, and denial of service attacks.

Vulnerabilities

Weak passwords, lack of input validation, and insufficient error handling.

Risks

Data exposure, unauthorized access to user accounts, and service downtime.

Controls

  • Input Validation: Ensuring that user inputs are validated to prevent SQL injection and other attacks.
  • Password Hashing: Using bcrypt to hash passwords before storing them in the database.
  • Session Management: Implementing secure session management to track user logins.
  • Logging: Keeping logs of user actions for auditing and non-repudiation.
